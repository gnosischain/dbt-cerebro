fct_mixpanel_ga_gpay_crossdomain_daily

Daily rollup of Gnosis Pay ↔ Mixpanel matching with role-bucket and activity dimensions. Privacy boundary: every cross-domain join uses the keyed pseudonym from the pseudonymize_address macro, applied identically on both sides. Raw EOAs and Safe addresses never appear in this output. Methodology: a per-user identity bridge (matched_pairs) reads the precomputed (user_id_hash, gp_safe, matched_roles) tuples directly from fct_mixpanel_ga_gpay_users so the JOIN is not recomputed here. Same gp_safe → activity flags (delay_active_safes_7d, allowance_changed_safes_30d) are computed once and joined into both rollups. mp_user_flags rolls up role flags and activity flags to one row per Mixpanel user; mp_daily then computes per-day matched-user counts sliced by role bucket and activity flag, and gpay_daily contributes the on-chain DAU comparison. Implementation note: explicit AS aliases on the mp.* columns in the final SELECT are required. Both mp_daily and gpay_daily expose a `date` column, so ClickHouse keeps the qualified name `mp.date` in the output projection and the config's order_by=(date) would otherwise fail to resolve. The non-ambiguous columns get explicit aliases too as a defensive consistency pass. Headline metrics: mp_dau — Mixpanel daily active users (production, all identification states; anonymous device users are counted but never match the GP side). onchain_active_users — distinct GP wallets with on-chain activity that day, from fct_execution_gpay_activity_daily. matched_users_any — DAU users matched on ANY identity role (initial_owner ∪ delegate ∪ safe_self). matched_by_* — same DAU users sliced by which role matched. users_with_delay_activity_7d — matched users whose gp_safe queued at least one TransactionAdded event in the last 7 days. users_with_allowance_changes_30d — matched users whose gp_safe had a SetAllowance event in the last 30 days. Backwards-compat: the original `matched_users` and `match_rate_pct` columns are preserved (now equal to matched_users_any and the corresponding rate) so existing dashboards keep working.
