int_revenue_ocsdai_user_balances_daily

OpenCover OC-sDAI ERC-4626 vault look-through for the sDAI revenue stream. Per (date, user) USD value of the sDAI underlying each OC-sDAI shareholder's position: `balance_usd = oc_shares * share_price * sDAI_usd_price`, where `oc_shares` is the per-holder OC-sDAI balance (int_execution_tokens_balances_native_daily), `share_price` is sDAI-per-share (int_yields_ocsdai_share_price_daily), and `sDAI_usd_price` comes from int_execution_token_prices_daily. Consumed by int_revenue_sdai_fees_daily as a third UNION branch alongside native sDAI and Aave aGnosDAI balances. The vault's own pooled sDAI is NOT double-counted: it is held under the vault address, which the sdai model's non_user_contracts anti-join drops (the vault is in tokens_whitelist). The vault's self-held OC-sDAI shares (async escrow) are dropped by the same downstream anti-join.
