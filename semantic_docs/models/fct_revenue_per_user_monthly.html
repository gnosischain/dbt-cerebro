fct_revenue_per_user_monthly

Monthly per-user revenue. Same shape as the weekly variant but at calendar-month grain with `month_fees_total` (no rolling) and the month-scaled $0.50 active threshold.

Module: revenue

Status: approved

Owner: analytics_team

Path: models/revenue/marts/fct_revenue_per_user_monthly.sql

Tags: production revenue revenue_cross

Columns

NameTypeDescription
has_gnosis_appUInt81 if user had Gnosis App-stream fees this month.
has_gpayUInt81 if user had Gnosis Pay-stream fees this month.
has_holdingsUInt81 if user had holdings-stream fees this month.
has_sdaiUInt81 if user had sDAI-stream fees this month.
is_revenue_activeUInt81 iff month_fees_total >= $0.50 (month-scaled $6 threshold).
monthDateFirst day of the calendar month.
month_fees_totalFloat64Sum of monthly potential revenue across all streams (USD).
n_streamsUInt8Number of distinct streams contributing in the month.
