int_execution_mmm_controls_weekly

Long-form weekly MMM control variables (macro / seasonality / operational signals the regression conditions on but does not attribute to). One row per (`week`, `control_name`) on the same continuous weekly spine as KPIs / media. Source aggregators (weekly grain): gno_usd_price_avg ← avg(price) over int_execution_token_prices_daily filtered to symbol='GNO' eth_usd_price_avg ← avg(price) where symbol IN ('WETH','ETH') wxdai_eur_proxy_avg ← avg(price) where symbol IN ('WXDAI','EURE') chain_gas_price_gwei_avg ← avg(gas_price_median) over int_execution_transactions_info_daily chain_block_count ← sum(n_txs) week_of_year ← toWeek(week) (computed) week_index ← weeks-since-spine-start (computed) is_holiday_week ← LEFT JOIN to seed mmm_holiday_weeks hardfork_step ← 1 from the fork-week onward via LEFT JOIN to seed mmm_hardfork_steps. Computed via LEFT JOIN to a `count of forks ≤ week` sub-CTE rather than a correlated subquery (ClickHouse plans correlated subqueries inefficiently and they OOM at scale). Incremental on `week` (monthly partition).
