fct_consensus_validators_explorer_daily

Per-operator (withdrawal_credentials) daily roll-up feeding the five daily charts on the Validator Explorer tab. Materialized as a physical table sorted on withdrawal_credentials so the API's `WHERE withdrawal_credentials = 'x'` filter prunes at read time via the primary index. Rebuilt incrementally month-by-month to stay under the cluster's 10.8 GiB memory cap during backfill. APY is balance-weighted (drops apy=0 idle validators): apy = SUMIf(apy * balance_prev_gno, 0 < apy < 200 AND balance_prev_gno > 0) / SUMIf(balance_prev_gno, 0 < apy < 200 AND balance_prev_gno > 0) falling back to 0 when no validators qualify. See api_consensus_validators_apy_mean_daily for the rationale. v3: drops post-exit rows. Without this, an exited validator keeps emitting zero-income / zero-balance rows from int_consensus_validators_income_daily forever, which the UI renders as a flat tail on the credential's balance line. Joining stg_consensus__time_helpers + status_latest and filtering `date <= genesis + exit_epoch * slots_per_epoch * seconds_per_slot` (or exit_epoch is FAR_FUTURE_EPOCH = 2^64-1, meaning "not exited") makes every chart downstream stop at the validator's actual exit.
