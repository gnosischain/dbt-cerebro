fct_execution_gnosis_app_retention_by_action_monthly

Cohort retention split by activity_kind (swap_filled / swap_signed / topup / marketplace_buy / token_offer_claim / circles_*). Same onboard-cohort anchor and denominator as fct_execution_gnosis_app_retention_monthly. Modeling assumptions: * cohort_month = the user's GLOBAL onboard month (first 'onboard' row) — the SAME anchor for every action, so denominators are comparable across action types. * users (numerator) = distinct cohort users who did THIS activity_kind in activity_month (lower triangle only, activity_month >= cohort_month). * initial_users (denominator) = onboard-cohort size = distinct users onboarded in cohort_month. NOT max(users) over months (peak-normalisation -> retention > 100%) and NOT the month-0 action adopters: because the numerator counts any cohort user doing the action each month (not a fixed followed sub-cohort), a later month can have more adopters than month 0, so only the full onboard cohort bounds the ratio at <= 100%. * retention_pct reads as "% of the onboard cohort doing this action in month N" (action adoption), not conditional stickiness. Small/rare actions therefore show low (correct) percentages, not ~100%.
