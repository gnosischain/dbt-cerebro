fct_execution_gnosis_app_journeys_60d

Pre-joined GA-side journey spine with a 60-day lookback. One row per (user_pseudonym, conversion_kind, conversion_ts, conversion_date, event_kind) group — one row per conversion × distinct prior-60d touchpoint event_kind (n_touches / first_touch_ts / last_touch_ts / td_sum measures), NOT one row per touch; the conversion's own event kind is excluded. Built via `build_journey_lookback(60, 'gnosis_app')` so the leakage guards (`event_ts < conversion_ts` and `event_kind != conversion_kind_to_event_kind(conversion_kind)`) are applied at materialisation, not at query time. The MTA persona reads this directly to score first/last/linear/ time-decay credit per touch without re-running the conversions × events self-join. The 7d / 14d / 60d sensitivity-sweep variants (deferred) reuse the same macro with a different `lookback_days` argument. Incremental, monthly partition on `conversion_date`. Uses `grace_hash` join algorithm to keep memory bounded across the explicit lookback window.
