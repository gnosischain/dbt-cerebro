int_ubo_second_level_daily

Rows from fct_ubo_supply_claims_daily where ubo_address is itself a known container for the bridge token (container_address). These are the "second- level" rows that fct_ubo_supply_claims_resolved_daily needs to redistribute. Materialized separately so the resolved model can join this tiny table on the right (hash-table) side while streaming the full claims dataset through on the left (probe) side, avoiding loading millions of rows into a hash table. Identical schema and unique key to fct_ubo_supply_claims_daily.
