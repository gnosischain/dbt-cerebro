fct_ubo_supply_claims_daily

THE reusable UBO supply-claims surface. One row per (date, protocol, container_address, ubo_address) — for each container contract that pools a token, the individual end-holders and their withdrawable balance. Phase 1 contributors: Aave V3 + SparkLend. Phase 2+ will add Balancer V2/V3, Curve, and V2-style AMMs via additional UNION ALL branches; downstream consumers do not change. Use this model to answer "who really holds this token" questions — join with int_execution_tokens_balances_daily on (token_address, address) to replace container-contract rows with their UBOs.
