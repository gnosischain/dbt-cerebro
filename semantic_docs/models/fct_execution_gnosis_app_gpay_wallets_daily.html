fct_execution_gnosis_app_gpay_wallets_daily

Daily cumulative count of Gnosis Pay wallets controlled by a Gnosis App user, split by onboarding class. Backing fact table for api_execution_gnosis_app_gpay_wallets_daily. onboarded_via_ga : the very first owner of the GP Safe was a GA user. User went from zero → Gnosis Pay via the Gnosis App. imported : GA user appears in the owner set after creation. User already had Gnosis Pay and later linked their GA. V1 tracks the date a GP wallet first gained a GA owner (first_ga_owner_at) and its cumulative count over time. Removals (GA user subsequently removed from the owner set) are NOT subtracted here — dashboards can read is_currently_ga_owned from int_execution_gpay_ga_wallets for a point-in-time currently-owned snapshot.
