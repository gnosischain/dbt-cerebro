int_execution_gnosis_app_user_identity_bridge

INTERNAL ONLY — holds raw `address` and `user_pseudonym` together for every known Gnosis App user. The only place in the warehouse where the (raw address, pseudonym) pairing exists. Other GA-side models that need to pseudonymize a raw address column at build time read from this bridge instead of recomputing the keyed hash. Privacy posture: - Tagged `internal_only` + `privacy:tier_internal`. - `meta.expose_to_mcp = false` so cerebro-mcp's discover_models filter hides the model entirely. - Has no `api:<resource>` tag, so cerebro-api's `_build_routes` never registers it as an endpoint. - cerebro-mcp also denies the bare table name from `execute_query` via the `INTERNAL_ONLY_TABLES` runtime guard. `pseudonymize_address` is deterministic with the same salt (`CEREBRO_PII_SALT`) used by `stg_mixpanel_ga__events`, so the `user_pseudonym` written here joins on equality to the `user_id_hash` column emitted at Mixpanel ingestion. One row per GA user (same row count as `int_execution_gnosis_app_users_current`).
