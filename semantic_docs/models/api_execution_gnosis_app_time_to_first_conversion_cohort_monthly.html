api_execution_gnosis_app_time_to_first_conversion_cohort_monthly

Time from onboard to a user's FIRST event of each conversion kind, bucketed by onboard-month cohort. Long format: one row per (cohort_month, conversion_kind) over topup / swap_filled / marketplace_buy / token_offer_claim. Built on int_execution_gnosis_app_first_conversion. Modeling assumptions: * cohort_month = toStartOfMonth(first_seen_at), the heuristic onboard month. The current (partial) month is excluded. * n_in_cohort = all users onboarded that month; n_converted = those with a non-NULL first_<kind>_at (a NULL first_*_at means the user never did that action). pct_converted = n_converted / n_in_cohort x 100. * median/p25/p75_days are over CONVERTERS ONLY, as dateDiff('day', first_seen_at, first_conversion_at), FLOORED AT 0: the heuristic onboard date can postdate a user's real first action (negative raw diff), and those users are reported as 0-day rather than negative.
