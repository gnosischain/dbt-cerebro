fct_execution_gnosis_app_funnel_daily

Per-day funnel diagnostics over `int_execution_gnosis_app_user_events_unified` using ClickHouse `windowFunnel`. Funnel definitions live in seed `mta_funnels.csv` (funnel_name + step_1/2/3 event_kinds + window_seconds). The model materialises one SELECT block per funnel via a Jinja loop over `run_query(seed)` and UNION ALLs the results — required because `windowFunnel(window_seconds)` needs `window_seconds` as a compile-time literal (a CROSS JOIN against the seed with the column is rejected by ClickHouse with BAD_ARGUMENTS at runtime). Each output row is one (date, funnel_name, user) triple with `level` = the max step reached by that user within `window_seconds` of their first matching event. `level=0` rows are emitted for users who hit step 1's event but not step 2 within the window. Adding a funnel: append a row to `mta_funnels.csv`, run `dbt seed --select mta_funnels`, then `dbt run` regenerates the model automatically.
