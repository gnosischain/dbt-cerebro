int_execution_gnosis_app_events_chain_unified

Long-form on-chain GA event log at full timestamp grain, keyed by `user_pseudonym` (no raw address ever leaves the bridge). Single source of chain-side touchpoints for MTA. UNION ALL of seven CTEs, each tagged with a distinct `event_kind` starting with `chain.`: chain.onboard — first-ever heuristic hit per user (from int_execution_gnosis_app_users_current) chain.<heuristic_kind> — seven heuristic rules from int_execution_gnosis_app_user_events (circles_register_human, circles_invite_human, circles_trust, circles_profile_update, circles_fee, circles_personal_mint, safe_invitation_module) chain.swap_signed — every PreSignature event in int_execution_gnosis_app_swaps chain.swap_filled — was_filled subset, ts = first_fill_at chain.topup — int_execution_gnosis_app_gpay_topups chain.marketplace_buy — int_execution_gnosis_app_marketplace_payments chain.token_offer_claim — int_execution_gnosis_app_token_offer_claims Incremental (monthly partition on `event_date`); reads user_pseudonym from `int_execution_gnosis_app_user_identity_bridge`. Use `grace_hash` join algorithm (set in pre-hook) to keep memory bounded across the seven UNIONs. `event_kind` is LowCardinality and validated against seed `mta_event_kinds.csv` (relationship test, severity warn).
