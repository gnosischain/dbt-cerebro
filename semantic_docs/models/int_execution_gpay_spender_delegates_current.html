int_execution_gpay_spender_delegates_current

Current spender-delegate set per Gnosis Pay Safe. Replays AssignRoles events from int_execution_gpay_roles_events with argMax per (roles_module, member, role_key) and keeps only the combinations whose latest `is_member` is 1. Joins back to the GP Safe via int_execution_gpay_safe_modules (RolesModule proxy → owning Safe). A Safe with a Roles module that has authorized 2 spender keys produces 2 rows here. A Safe with no delegates produces 0 rows. The delegate address is the EOA the GP card-flow infrastructure uses to trigger spends within the user's allowance.
