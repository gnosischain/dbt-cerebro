stg_live__dex_trades_balancer_v2

Normalized Balancer V2 Swap events from the Vault contract (execution_live). PoolId → pool_address is resolved via the materialized `stg_pools__balancer_v2_pool_registry` table (built from historical PoolRegistered events). Using a pre-materialized registry keeps this view cheap to query from the dashboard; re-scanning the full historical Vault events on every poll would be prohibitive.
