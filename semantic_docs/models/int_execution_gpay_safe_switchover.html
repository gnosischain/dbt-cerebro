int_execution_gpay_safe_switchover

One row per June 2026 migrated pair. Refunded (is_lost) pairs are excluded from user holdings from first_refund_at (the first refund from the verified distributor 0xbb897a56...); non-refunded pairs have no cutover - their old safe counts until the user moves the funds. is_lost + first_refund_at are the reusable lost-safes filter; new_safe_deployed_at is informational.
