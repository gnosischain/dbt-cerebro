fct_consensus_validators_withdrawal_addresses_distinct

One row per `user_pseudonym` (sipHash of validator withdrawal address). Aggregates `n_validators_controlled` per address and bucketises into concentration tiers (single / small / medium / large / whale). `user_pseudonym` is the canonical cross-sector join key (sipHash64 of lowercased address with the project PII salt). Same hash space as fct_revenue_per_user_*, fct_execution_gpay_users_distinct, fct_execution_gnosis_app_users_distinct, fct_execution_circles_human_avatars_distinct, and Mixpanel `user_id_hash`. Excludes 0x00-type BLS-credential validators (their credentials are BLS pubkey hashes, not EVM addresses — can't be projected into the pseudonym space). See model description for the full caveat list.
