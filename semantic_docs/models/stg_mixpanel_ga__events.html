stg_mixpanel_ga__events

Incremental staging table over raw Mixpanel events tracking Gnosis App (app.gnosis.io) user interactions. Extracts all useful JSON properties into typed columns with privacy transformations applied: distinct_id and device_id are replaced with keyed pseudonyms via macros/pseudonymize_address.sql (salted with env_var CEREBRO_PII_SALT, so the hash is not reversible via rainbow tables against the public on-chain address space), city-level geo is excluded, and full URLs are reduced to path-only. Materialized as a table (not a view) so downstream intermediate models read lightweight typed columns instead of re-parsing the heavy JSON properties blob on every query. Structure: a `raw` CTE extracts all JSON properties and computes boolean flags. event_category is computed in the outer SELECT so it can reference is_autocapture (a derived column) without a self-join. To change classifications, edit macros/mixpanel_ga_event_category.sql. Privacy notes: the keyed pseudonym macro is salted so hashes cannot be reversed via rainbow tables against the public on-chain address space. The same macro must be used on on-chain addresses for any cross-domain join. The is_production flag identifies events from app.gnosis.io (~89% of traffic; the remainder is deploy previews and localhost). New columns added for domain context: is_identified — 1 if the user connected a wallet (distinct_id != '$device:*') is_production — 1 if the event originated from app.gnosis.io (not preview/localhost) event_category — derived classification via macros/mixpanel_ga_event_category.sql event_amount, event_sku, event_seller, event_value, event_asset_id — feature-specific custom properties for Circles/Marketplace/Swap events

is_productionUInt81 if the event originated from app.gnosis.io; 0 for deploy previews (deploy-preview-*.app.gnosis.io), localhost, or kitchen.app.gnosis.io. Approximately 11% of raw traffic is non-production. All intermediate models filter to is_production = 1.

user_id_hashUInt64Keyed pseudonym of distinct_id (sipHash64 over a salted + lowercased input; salt from env_var CEREBRO_PII_SALT). Non-reversible without the salt.
