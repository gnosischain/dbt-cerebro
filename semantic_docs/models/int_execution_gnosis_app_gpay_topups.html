int_execution_gnosis_app_gpay_topups

Gnosis App → Gnosis Pay TopUps. A TopUp = a CoW Protocol trade initiated by a GA user whose bought token ends up in a GP wallet in the same settlement tx. Detection: 1. Filter int_execution_cow_trades to taker ∈ GA users. 2. In the same transaction_hash, find an ERC-20 Transfer of the trade's token_bought_address whose recipient is a known GP wallet. Per design decision (April 2026), we do NOT filter by token_sold = CRC — the original Slack heuristic suggested CRC-only but the Dune dashboard already shows TopUps in many tokens. We expose token_sold_symbol so the dashboard can filter client-side. Scope note: uses int_execution_transfers_whitelisted_raw for the payout side, which only covers whitelisted tokens. The vast majority of GP top-ups are into EURe / USDC (whitelisted), so this covers the practical use-case. TopUps into non-whitelisted tokens would be missed — rare and acceptable for V1.
