int_execution_gnosis_app_conversions

GA conversion registry — one row per observed conversion event, with `conversion_kind` carried as a column rather than a separate table per kind. The MTA persona filters with `WHERE conversion_kind = '<kind>'` to swap conversion targets without changing SQL shape. Sources (UNION ALL): topup ← int_execution_gnosis_app_gpay_topups swap_filled ← int_execution_gnosis_app_swaps WHERE was_filled token_offer_claim ← int_execution_gnosis_app_token_offer_claims marketplace_buy ← int_execution_gnosis_app_marketplace_payments Each source INNER JOINs the identity bridge on lower(raw address) → `user_pseudonym`, so this model never holds a raw address. Incremental on `conversion_date` (monthly partition). The dedup key `conversion_dedup_key` is a cityHash64 of the kind + the smallest natural unique key per source (transaction_hash + log_index for most; order_uid for swaps).
