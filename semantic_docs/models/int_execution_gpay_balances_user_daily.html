int_execution_gpay_balances_user_daily

User-holdings view over int_execution_gpay_balances_daily that is safe to aggregate across the June 2026 Safe migration without double counting: refunded (is_lost) pairs' old-safe rows count only before first_refund_at; non-refunded pairs have no cutover - their old safe counts until the user moves the funds. All user-holdings consumers (balance aggregates and fct_execution_gpay_user_balances_latest) read this view; raw on-chain per-Safe balances remain in int_execution_gpay_balances_daily.
