int_execution_gnosis_app_user_events

Long-form heuristic event log for Gnosis App user identification. Six rules, all keyed off the same chokepoint: a Cometh ERC-4337 bundler (one of three relayer addresses in seeds/gnosis_app_relayers.csv) sent the outer transaction to the canonical EntryPoint v0.7. Any account that appears as a parameter inside such a transaction is, by construction, acting through Gnosis App for that action. Rules: safe_invitation_module — Safe created in the same Cometh tx that enabled the Circles InvitationModule (0x00738aca…). The Safe address is the user. circles_metri_fee — CRC ERC-1155 transfer to the Metri fee receiver (0x97FD8f78…) inside a Cometh tx. circles_register_human — Hub.RegisterHuman event inside a Cometh tx. decoded_params['avatar'] is the user. circles_invite_human — Same RegisterHuman event with a non-zero decoded_params['inviter'] — the inviter is the user. circles_trust — Hub.Trust event inside a Cometh tx. decoded_params['truster'] is the user. circles_profile_update — NameRegistry.UpdateMetadataDigest event inside a Cometh tx. decoded_params['avatar'] is the user. Address columns are stored lowercase WITH the 0x prefix to match the rest of the dbt outputs and the format that decode_logs produces for decoded_params address values. Rule 1 reads from int_execution_safes (which is already 0x-prefixed) and rules 2-6 read from contracts_circles_v2_*_events.decoded_params (also 0x-prefixed). Privacy: this is the source-of-truth intermediate for sector membership. Raw addresses are kept here because they are required for the snapshot argMax and for the cross-reference with int_execution_safes. The pseudonymization boundary is in int_execution_gnosis_app_user_identities, one model downstream.

addressStringThe Gnosis App user address (avatar / inviter / truster / Safe, depending on the rule). Lowercase, 0x-prefixed — the same form that decode_logs writes for decoded_params address values and that stg_mixpanel_ga__events sees for wallet-identified users. Storing in this form means the cross-domain pseudonym join lines up exactly without any per-side normalization.
