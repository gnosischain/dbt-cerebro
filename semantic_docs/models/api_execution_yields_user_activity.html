api_execution_yields_user_activity

Per-event user activity feed across LP and lending positions (deposits, withdrawals, fee collections, supply/repay, etc.). One row per on-chain action. Sourced from int_execution_yields_user_activity.

Module: execution

Status: candidate

Owner: analytics_team

Path: models/execution/yields/marts/api_execution_yields_user_activity.sql

Tags: production execution yields api:yields_user_activity granularity:latest tier1

Columns

NameTypeDescription
actionStringAction label (e.g. 'Deposit', 'Withdraw', 'CollectFees', 'Supply', 'Repay').
amountFloat64Action amount in native units.
amount_usdNullable(Float64)Action amount in USD (NULL when pricing is unavailable).
block_timestampDateTimeUTC timestamp of the on-chain LP or lending event.
dateDateCalendar day of the event (derived from block_timestamp).
position_addressStringPool or reserve address depending on the action type.
protocolStringProtocol name — a DEX (Uniswap V3, Swapr V3, Balancer) for LP events or a lending market (Aave V3, SparkLend) for lending events.
sourceStringSource model identifier — distinguishes LP vs lending events.
token_addressStringToken address of the amount moved (for lending, the reserve address).
token_symbolStringToken symbol of the amount moved.
transaction_hashStringTransaction hash of the event.
wallet_addressStringWallet address that performed the action.
