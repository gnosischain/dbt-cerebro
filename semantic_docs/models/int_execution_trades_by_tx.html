int_execution_trades_by_tx

Transaction-grain collapse of `int_execution_pools_dex_trades` — one row per (date, transaction_hash). Runs the expensive tx-level GROUP BY and `int_crawlers_data_labels` JOIN exactly once, so downstream facts and api views operate on a small pre-shaped table instead of scanning the 31M+ swap rows. `aggregator_label` uses the 3-bucket Option-2 heuristic: labeled → project name; unlabeled + multi-hop → "Other Router"; unlabeled + 1-hop → "Direct". `hop_bucket` and `size_bucket` are pre-computed so api views can group by string columns directly.
