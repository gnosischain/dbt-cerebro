int_execution_gpay_allowances_current

Current allowance state per Gnosis Pay Safe. Replays SetAllowance events from int_execution_gpay_roles_events and keeps the latest row per (roles_module, allowance_key). Joins back to the GP Safe via int_execution_gpay_safe_modules. Columns are projected as Strings because the underlying decoded_params values are Strings (decode_logs returns Map(String, String)). Cast to numeric types in downstream marts when needed — keeping them as Strings here avoids precision loss for uint128 balance values that exceed UInt64. `refill` is the canonical "daily limit" number — the amount the allowance refills by once every `period` seconds.
