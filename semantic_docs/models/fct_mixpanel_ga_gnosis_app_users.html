fct_mixpanel_ga_gnosis_app_users

Per-user fact for the Gnosis App sector with Mixpanel as a CHECK, not as a source of truth. Every row from int_execution_gnosis_app_user_identities appears here. matched_mp is a column that flags whether Mixpanel has at least one identified production event from the same pseudonym — it is NOT a join filter. The heuristic set is authoritative; Mixpanel coverage is diagnostic. This is the opposite of fct_mixpanel_ga_gpay_users (which uses INNER JOIN against the GP identity model because GP has a clean upstream membership label list). Both bridge patterns coexist by design.
