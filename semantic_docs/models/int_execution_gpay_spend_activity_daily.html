int_execution_gpay_spend_activity_daily

Daily card-spend activity per GP Safe. Source: int_execution_gpay_spender_events filtered to event_name='Spend'. The Spend event already carries `account` (the debited Safe), so we don't need to join through int_execution_gpay_safe_modules — the account address IS the gp_safe. We INNER JOIN to int_execution_gpay_wallets to guard against any cross-Safe Spend events that wouldn't be GP-related. Aggregates count of spends and number of distinct assets/receivers. Amounts are not summed because they're per-asset and per-decimals; a follow-up model can do the USD valuation join.
