int_execution_gnosis_app_token_offers

Gnosis App token-offer registry — one row per `NextOffer` instance emitted by the Circles v2 ERC20TokenOfferCycle contracts. Each cycle (there are 3 deployed — v1/v2/v3) periodically emits `NextOfferCreated` for a new sale instance with a fixed CRC price and per-user CRC cap; users then pay CRC to receive the cycle's `offerToken` (GNO today). Sources & join: 1. CycleConfiguration event on the cycle address — carries offerToken, admin, offersStart, offerDuration. 2. NextOfferCreated event on the same cycle address — carries the specific offer (nextOffer) address, tokenPriceInCRC, offerLimitInCRC. 3. Joined on (contract_address = cycle_address) so each offer is enriched with its parent cycle's configuration. 4. NextOfferTokensDeposited aggregated by offer for supply side. Token metadata (symbol, decimals) comes from tokens_whitelist, keyed on the lowercase offer_token_address.
