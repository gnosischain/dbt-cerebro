int_execution_gnosis_app_user_identities

Pseudonymization boundary for the Gnosis App sector. Every address from the snapshot is hashed via pseudonymize_address before flowing into any marts model. Upstream addresses are already 0x-prefixed (the convention used everywhere in the dbt outputs and the same form stg_mixpanel_ga__events ingests for wallet-identified Mixpanel distinct_ids), so the hash on both sides of the cross-domain join is computed over identical inputs. Output: one row per Gnosis App user pseudonym, joinable to stg_mixpanel_ga__events.user_id_hash on equality. No raw_address column is exposed — for v1 we drop it entirely. If a later cross-product join (e.g. GP × Gnosis App) needs raw addresses for an internal-only intermediate, that join lives upstream of this model, not downstream.
