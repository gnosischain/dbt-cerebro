api_execution_live_trades_freshness

One-row freshness indicator for the live-trades tab. Shows the newest ingested block timestamp in `execution_live.logs`, the server clock, and the lag in seconds. Intended for a "Data as of X, lag Y min" UI element — important because the tab is anchored on ingested-data time, not wall clock. Reads directly from the source so it keeps working when the feed itself is empty.
