fct_execution_address_resolver

Per-(address × source) domain resolver rows. The cross-source merge happens at query time inside `api_execution_address_resolver` (point lookup with `WHERE address = '0x…'` + `GROUP BY address`) instead of at build time, because a 5–10M-address outer GROUP BY OOMs the cluster cap. AggregatingMergeTree on `address` lets background merges collapse duplicate rows when address is in only one source; the api view handles the rest. Materialised as a table (not a view) so the Account Portfolio custom view's per-request lookup is an O(log N) primary-index seek rather than a 5-way UNION scan. Expected cardinality ~500k-1M on Gnosis Chain. display_name precedence: Circles avatar name → role tag ("Validator operator · N validators" / "Safe owner · N safes" / "Safe contract" / "Gnosis Pay wallet" / "Circles avatar") → empty. LabelSelector filters on substring match so both hex paste and name search resolve from the same field.
