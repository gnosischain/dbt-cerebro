int_execution_gnosis_app_first_conversion

One row per Gnosis App user (the onboard cohort) with the date of their first event in each conversion kind. Feeds the time-to-first-conversion view. Modeling assumptions: * first_seen_at = the heuristic onboard date, carried from int_execution_gnosis_app_user_activity_daily ('onboard' rows). * Each first_<kind>_at is NULL when the user never did that action. This is enforced via nullIf(minIf(...), toDate('1970-01-01')) plus a join_use_nulls hook, so the ClickHouse Date default (1970-01-01) from an empty minIf or an unmatched LEFT JOIN cannot masquerade as a real conversion. Consumers MUST treat 1970-01-01 as impossible and rely on IS NULL (a prior bug let the epoch sentinel inflate pct_converted to 100% and produce negative time-to-conversion). * Conversion kinds: topup, swap_filled, marketplace_buy, token_offer_claim.
