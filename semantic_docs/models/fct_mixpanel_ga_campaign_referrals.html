fct_mixpanel_ga_campaign_referrals

Campaign REFERRAL composition (state-based snapshot) — of each first-touch campaign's identified users, how many became Circles inviters and how many people they brought in. Two scopes mirroring fct_execution_gnosis_app_gt_referrals (REF-D01): earned (GA-native paid-reward ledger) and full_graph (every accepted invite, avatar.invited_by — a ~2.2x superset). invites_* count DISTINCT invitees, not ledger edges. Inviter addresses join user_id_hash via the shared salted pseudonym (identified Mixpanel distinct_id == on-chain address) — no raw address materializes. Outcome-based: an inviter only counts once an invite is ACCEPTED on-chain, so counts run LOWER than Mixpanel-side "inviters" tiles that count invite-link sharing (Growth dashboard 'referral': 152 sharers vs 114 full-graph accepted inviters, 2026-07). K-anonymity: campaigns with < 5 all-time signups bucketed into '_small_campaigns'. Aggregate-only output.
