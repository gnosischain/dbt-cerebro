fct_execution_gnosis_app_journeys_30d

Pre-joined GA-side journey spine with a 30-day lookback. One row per (conversion, prior-30d touchpoint) pair with the conversion's own event kind excluded from touches. Built via `build_journey_lookback(30, 'gnosis_app')` so the leakage guards (`event_ts < conversion_ts` and `event_kind != conversion_kind_to_event_kind(conversion_kind)`) are applied at materialisation, not at query time. The MTA persona reads this directly to score first/last/linear/ time-decay credit per touch without re-running the conversions × events self-join. The 7d / 14d / 60d sensitivity-sweep variants (deferred) reuse the same macro with a different `lookback_days` argument. Incremental, monthly partition on `conversion_date`. Uses `grace_hash` join algorithm to keep memory bounded across the explicit lookback window.
