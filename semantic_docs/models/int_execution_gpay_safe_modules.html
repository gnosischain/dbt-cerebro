int_execution_gpay_safe_modules

Current module topology per Gnosis Pay Safe. Replays int_execution_safes_module_events filtered to GP Safes, keeps the latest event per (safe, module_proxy) via argMax, and drops rows whose latest event is `disabled_module`. The remaining rows are joined to contracts_gpay_modules_registry to get the module type (DelayModule / RolesModule / SpenderModule). Expected row count: ≈3 × |int_execution_gpay_wallets|. Anything other than 3 per Safe indicates a partially-onboarded GP Safe (worth investigating).
