int_execution_gnosis_app_user_activity_daily

Foundation table for all Gnosis App user-activity analytics — one row per (date, address, activity_kind). Every downstream cohort, retention, churn, DAU/WAU/MAU, and activity-by-action mart derives from this table. activity_kind enum: onboard — first-ever heuristic hit for this address (from int_execution_gnosis_app_users_current. first_seen_at). Exactly one row per user. Anchors the cohort_month for retention. circles_register_human, circles_invite_human, circles_trust, circles_profile_update, circles_metri_fee, safe_invitation_module — raw heuristic rows from int_execution_gnosis_app_user_events. swap_signed — PreSignature events (all rows in int_execution_gnosis_app_swaps). swap_filled — subset with was_filled = true. topup — GA→GP TopUps (int_execution_gnosis_app_gpay_topups). marketplace_buy — GA marketplace purchases (int_execution_gnosis_app_marketplace_payments). token_offer_claim — GA token-offer claims (e.g. CRC → GNO discount) (int_execution_gnosis_app_token_offer_claims). amount_usd populated for swap_filled, topup, and token_offer_claim (USD on the *received* side only). marketplace buys carry NULL until the Circles token price feed is wired.
