int_execution_transactions_by_project_daily

This model aggregates daily execution transaction data by project, providing insights into transaction volume, user engagement, and fees associated with each project over time.

Module: execution

Status: candidate

Owner: analytics_team

Path: models/execution/transactions/intermediate/int_execution_transactions_by_project_daily.sql

Tags: production execution transactions

Columns

NameTypeDescription
addressStringThe blockchain address associated with the crawler label, used for linking transaction data to project labels.
dateDateTimeThe date of the transactions, formatted as DateTime.
fee_native_sumFloat64Total fees paid in native tokens (wei converted to GNO/XDAI).
gas_used_sumFloat64Sum of gas used across all transactions, in native units.
projectStringThe name or identifier of the project associated with the transaction, defaulting to 'Unknown' if not specified.
sectorStringThe business sector or category assigned to the project, with 'Unknown' used when sector information is missing.
tx_countUInt64Total number of transactions for the project on the given date.
ua_bitmap_stateStringBitmap state representing unique addresses involved in transactions.
