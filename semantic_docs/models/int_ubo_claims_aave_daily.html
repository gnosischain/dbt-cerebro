int_ubo_claims_aave_daily

Per-protocol UBO supply claims for Aave V3 + SparkLend. Standardized shape: one row per (date, protocol, container_address, ubo_address) capturing the individual end-holder's withdrawable balance in the pool. Sourced from int_execution_lending_aave_user_balances_daily joined to lending_market_mapping for the container (aToken) address. Materialized daily so it can be re-used cheaply by any consumer. token_address is the canonical token contract per tokens_whitelist (symbol + date range), NOT the raw Aave reserve address. Aave pool contracts are immutable so they always reference the original reserve address even after token migrations (e.g. EURe V1 0xcb44 → V2 0x420c on 2024-08-25). The canonical lookup ensures UBO claims align with int_execution_tokens_balances_daily, which also normalises via tokens_whitelist.
