api_mixpanel_ga_gpay_acquisition_weekly

Not exposed via cerebro-api; accessible to the internal MCP / semantic layer. Aggregate-only pass-through view over fct_mixpanel_ga_gpay_acquisition_weekly (campaign-level counts only, no user_pseudonym / user_id_hash). Weekly first-funded accounts and first card transactions by UTM campaign, for the growth team. attribution_model selects the window: 'first_touch' = campaign on the account's earliest utm-tagged Mixpanel event, 'last_touch' = latest (derived per user by ordering events on event_time). UTM is sparse (~3% of events, entry/landing only); unattributed accounts are 'unknown'. Both windows are stacked long — filter attribution_model to one value before summing new_accounts.
