int_celo_gpay_wallets

Canonical Celo GP card Safe list, reconstructed from int_celo_gpay_wallet_events (action='issued_at') rather than read from a per-safe snapshot table — see sources.yml for why the earlier mutable-row wallets table was replaced with an append-only event log. first_spend_at/is_activated are derived here directly from crawlers_data.celo_gpay_transfers (not from an ingested event) — see this model's own header comment for why that read goes straight to the source rather than through int_celo_gpay_activity (would create a circular dbt ref) and why a full scan of the transfers table on every rebuild is acceptable (Gnosis Chain's own int_execution_gpay_wallets already does an equivalent full scan of the much larger execution.logs, successfully, as a materialized='table' full rebuild). Deliberately materialized='table' (full rebuild), not incremental, and expected to stay that way indefinitely: this table's size is bounded by total card count, not transaction volume, so it stays small even at full scale. A full rebuild also correctly re-checks is_activated/first_spend_at for every Safe on every run regardless of how long ago it was issued — e.g. a Safe issued today whose first spend happens two months from now is picked up correctly with no special handling, because nothing here is windowed by the Safe's age.

Columns

NameTypeDescription
first_spend_atNullable(Date)Date of first spend (transfer to settlement). Null if never activated.
is_activatedUInt81 if the Safe has ever spent, else 0. Computed as first_spend_at IS NOT NULL — not stored anywhere, since it's a pure derivation with no interpretive logic, unlike transfer classification.
issued_atDateDate of card issuance (SafeSetup event, or module-enable time as fallback).
owner_addressNullable(String)SafeSetup owner / signer key (first owner encountered). For MiniPay cards, the MiniPay EOA — holds no funds. Nullable: a Safe can be recognized via the bridge-module or activation signal alone, before its SafeSetup event has been decoded, in which case this is genuinely unknown rather than defaulted to an empty string.
ownersArray(String)Full owner/signer list for the Safe (usually a single element). Empty array — not null — when SafeSetup hasn't been decoded yet, since "zero owners" is an unambiguous representation of "no data" for an array (unlike an empty string, which could be misread as a real address).
safe_addressStringGP card Safe address on Celo (0x-prefixed, lowercase). The fund holder.
