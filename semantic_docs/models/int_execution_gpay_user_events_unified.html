int_execution_gpay_user_events_unified

Long-form GP-side chain event log keyed by (`user_pseudonym`, `identity_role`). Source: `int_execution_gpay_activity` joined to the GP identity bridge on the lowercased Safe address — every chain event on a GP Safe fan-outs to all (pseudonym, role) pairs tied to that Safe. A payment from a 2-owner Safe yields 3 rows: `safe_self` + two `initial_owner`s. `event_kind` is derived from `int_execution_gpay_activity.action` via a `multiIf`: Payment → gp.payment Cashback → gp.cashback_claim Fiat Top Up → gp.deposit Crypto Deposit → gp.deposit Fiat Off-ramp → gp.withdrawal Crypto Withdrawal → gp.withdrawal Reversal → gp.payment <anything else> → gp.action_other v1 is chain-only. Cross-domain Mixpanel touchpoints on a GP Safe's owner are reached by joining this table to `int_execution_gnosis_app_user_events_unified` on `user_pseudonym` — same hash space across sectors. `event_dedup_key` excludes `log_index` because `int_execution_gpay_activity` doesn't carry one; the dedup tuple is (transaction_hash, token_address, counterparty, direction, identity_role).
