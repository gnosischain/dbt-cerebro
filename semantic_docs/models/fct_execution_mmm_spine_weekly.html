fct_execution_mmm_spine_weekly

MMM weekly spine — wide pivot of `int_execution_mmm_kpis_weekly`, `int_execution_mmm_media_weekly`, and `int_execution_mmm_controls_weekly`. One row per `week`, one column per (kpi | media | control) name, populated by `sumIf` against the long-form intermediates. Continuous time — every week within the spine range exists, even when no KPI / media / control rows reference it. This is the single mart the MMM analyst's runtime mapping points at. The persona reads it directly and runs adstock decay, Hill / log-log response-curve fits, and contribution decomposition against the `media_*` columns conditioned on `kpi_*` and `ctrl_*`. Adding a new column: update the corresponding intermediate's UNION, update the registry seed, then add a `sumIf` clause here for the new name. Schema is wide-by-design and bounded by the registry seeds (~30 cols today).
