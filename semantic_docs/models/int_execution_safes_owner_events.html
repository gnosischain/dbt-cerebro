int_execution_safes_owner_events

Long-form ownership-event log for every Safe in contracts_safe_registry. All ABI decoding is delegated to decode_logs(...) so no byte-offset arithmetic lives in this repo. The wrapping model only: • unrolls SafeSetup.owners[] into one row per initial owner • normalizes AddedOwner / RemovedOwner / ChangedThreshold into the same (event_kind, owner, threshold) shape IMPORTANT — incremental filtering: decode_logs handles its own var('start_month') / var('end_month') window AND its own `` max-timestamp incremental predicate. This model must NOT add another WHERE on block_timestamp — doing so would shadow decode_logs's filter and break incremental runs. The only thing the outer model mirrors from the batch pattern is the config-level incremental_strategy switch. Decoded array shape: decode_logs emits arrays as toJSONString([...]) under the param name, so decoded_params['owners'] is a JSON array string like '["0xabc","0xdef"]'. ClickHouse JSONExtract* functions are 1-indexed. SafeSetup unrolling uses a pre-filter subquery before the ARRAY JOIN because ClickHouse evaluates ARRAY JOIN's range() expression before the outer WHERE. Without the subquery, non-SafeSetup rows would reach the ARRAY JOIN with NULL `owners` and `range(1, JSONLength(NULL)+1)` would fail with `Illegal (null) value` — same fix pattern as int_execution_gpay_roles_events.assign_role_rows.
