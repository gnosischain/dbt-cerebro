fct_execution_circles_human_avatars_distinct

One row per `user_pseudonym` for Circles v2 HUMAN avatars (Groups and Orgs excluded — those aren't user-grain entities). `user_pseudonym` is the canonical cross-sector join key (sipHash64 of the avatar address with the project PII salt). Same hash space as fct_revenue_per_user_*, fct_execution_gpay_users_distinct, fct_execution_gnosis_app_users_distinct, and Mixpanel `user_id_hash`. Powers user-overlap analysis: "Circles humans who are also revenue- active", "Circles humans who hold a Gnosis Pay Safe", etc. Carries lightweight engagement / metadata flags so analysts can slice by registration path or profile-completion stage without re-joining to the metadata table.
