fct_mixpanel_ga_gpay_users

Per-user fact joining Mixpanel identified users to Gnosis Pay Safes through the keyed-pseudonym identity bridge. One row per (user_id_hash, gp_safe) pair where the user matched on at least one of the three identity roles (initial_owner, delegate, safe_self). Denormalizes the per-Safe module topology, current daily limit, and recent delay activity onto each row so downstream segment queries don't need extra joins. Privacy invariant: every column representing a person is either user_id_hash (already pseudonymized in stg_mixpanel_ga__events) or a Safe address (smart account, treated as non-PII). No raw EOA appears. Implementation note: explicit AS aliases on the m.* columns in the final SELECT are required. ClickHouse keeps the qualified name `m.gp_safe` in the output projection whenever the bare name `gp_safe` is ambiguous across joined relations (modules_per_safe, allowance_per_safe, delay_30d, spends_30d all expose a `gp_safe` column), and the config's order_by=(user_id_hash, gp_safe) would then fail to resolve.
