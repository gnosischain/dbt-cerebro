int_ubo_claims_uniswap_v3_daily

Per-user UBO supply claims for Uniswap V3. Standardized shape: one row per (date, container_address, ubo_address, token_address) capturing each LP's net withdrawable token balance in a Uniswap V3 pool. Unlike Balancer V2 (single Vault), each Uniswap V3 pool is its own token custodian so container_address is the individual pool address. Two tracks of LP attribution are merged: - Track A (direct LPs, ~45%): the pool Mint/Burn event's owner field is the real LP — used directly from stg_pools__dex_liquidity_uniswap_v3. - Track B (via NPM, ~55%): the pool event's owner is the NonfungiblePositionManager (0xae8fbe...). The real LP is resolved by decoding NPM Transfer events (ERC-721 ownership per tokenId) and joining NPM IncreaseLiquidity/DecreaseLiquidity to pool events via transaction_hash. Position transfers (LP A mints NFT → transfers to B → B burns) are handled by the Transfer event ownership chain. Balances are keyed by (ubo_address, symbol, container_address) through the cumsum so token migrations collapse into a single series.
