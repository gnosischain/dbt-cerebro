fct_execution_gnosis_app_attribution_30d

Pre-computed first / last / linear / time-decay (half-life = 7d) attribution per (`conversion_kind`, `event_kind`) over the last 180 days of conversions. The MTA persona reads this as the attribution-comparison table; a single `WHERE conversion_kind = '<kind>'` filter swaps in the desired conversion target. Implementation: pre-aggregate per conversion → tie-counts → per-touch score → group by `event_kind`. Pure GROUP BY + LEFT JOIN; no `groupArray + ARRAY JOIN` and no window functions. The pattern survived three rewrites — v1 arrays and v2 windows both OOM'd at the cluster's 10 GiB per-query cap on the 12 M-row journey set; v3 drops sort space from 12 M down to ~22 k per-conversion rows. Time-decay weight per touch: `td_raw = exp(-lag_days / 7)`, normalized so `Σ td_credit_per_conversion = 1.0`. First / last credit splits across ties so `Σ first_touch_credit_per_conv = 1.0` and `Σ last_touch_credit_per_conv = 1.0` exactly.
