int_execution_gpay_coverage_daily

Per-day, per-(`conversion_kind`, `identity_role`) tracked-coverage stats. Same shape as `int_execution_gnosis_app_coverage_daily` with the GP role dimension added so persona can compare owner-grain coverage against treasury-grain coverage for the same conversion kind. "Tracked": ≥1 row in `int_execution_gpay_user_events_unified` where `event_ts < conversion_ts`, `event_ts >= conversion_ts - 30d`, `identity_role` matches the conversion's role, and `event_kind` is not the conversion's own mapped event_kind (leakage guard via `conversion_kind_to_event_kind`). Incremental, monthly partition on `conversion_date`. Uses `grace_hash` join algorithm in pre-hook.
