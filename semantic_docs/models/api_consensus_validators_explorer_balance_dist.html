api_consensus_validators_explorer_balance_dist

Balance-distribution histogram across co-validators sharing a withdrawal credential. Light view over fct_consensus_validators_status_latest (~558k rows); the API prunes by withdrawal_credentials before the bucketing runs so the query is cheap even without a physical index on credentials. Bucket edges (<1, 1-16, 16-32, 32-48, 48-64, 64-128, 128-256, >=256 GNO) are chosen to span the two MaxEB regimes: 0x00 / 0x01 credentials cap at 32 GNO effective balance (pre-Pectra), while 0x02 compounders + EIP-7251 go up to 2048 GNO post-Pectra. The 16 / 32 / 64 / 128 / 256 edges surface clusters at both caps plus any partial-balance validators (withdrawing or slashed). bucket_order (0-7) is emitted to keep the API sort stable in ascending balance order.
