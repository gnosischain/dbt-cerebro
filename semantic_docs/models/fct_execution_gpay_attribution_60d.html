fct_execution_gpay_attribution_60d

Pre-computed first / last / linear / time-decay (HL=7d) attribution per (`conversion_kind`, `identity_role`, `event_kind`) over the last 180 days of conversions, using a 60-day touchpoint lookback. Same pre-aggregate-then-JOIN pattern as the 30d mart. The time-decay half-life is fixed at 7 days regardless of lookback — it is a customer-decay parameter, not a window parameter.
