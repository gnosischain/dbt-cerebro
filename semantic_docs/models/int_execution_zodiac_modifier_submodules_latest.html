int_execution_zodiac_modifier_submodules_latest

Latest-state snapshot of which sub-modules are currently enabled on each Zodiac Modifier, derived from int_execution_zodiac_modifier_module_events via argMax(event_kind) — keeps only (modifier, submodule) pairs whose most recent event is an enable. Full history lives in the events model. Generic Modifier -> sub-module relationship table. Classifies each sub-module: submodule_is_safe is 1 when the sub-module is itself a Safe (join to contracts_safe_registry). is_erc1271_exploitable mirrors that flag — the ERC-1271 revert-data bypass in the shared Modifier mastercopies is only reachable when the enabled sub-module is a Safe (an EOA sub-module cannot be forced to return 0x1626ba7e). The avatar Safe (the Safe that has the Modifier enabled) is resolved from the Safe-side module events; is_gp flags whether that avatar is a Gnosis Pay wallet.
