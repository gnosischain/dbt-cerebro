int_ubo_claims_balancer_v2_daily

Per-protocol UBO supply claims for Balancer V2. Standardized shape: one row per (date, container_address, ubo_address, token_address) capturing each LP's net withdrawable token balance in the Balancer Vault. Balancer V2 uses a single Vault contract (0xba12...2c8) that custodies all pool tokens. LPs interact with individual pool contracts but the token custody sits in the Vault, so container_address is the Vault for every row. Balances are computed as a daily cumulative sum of PoolBalanceChanged event deltas from stg_pools__dex_liquidity_balancer_v2, keyed by (ubo_address, symbol) so that token migrations (e.g. EURe V1 → V2) collapse into a single series. token_address is the canonical contract per tokens_whitelist (symbol + date range) on each balance date. Only whitelisted tokens (tokens_whitelist) are tracked. Positions with balance_raw <= 0 are excluded from output.
