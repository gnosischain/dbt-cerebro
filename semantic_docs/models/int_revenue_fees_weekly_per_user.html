int_revenue_fees_weekly_per_user

Weekly per-user fees per stream/symbol with trailing 52-calendar-week rolling total (`annual_rolling_fees`). The weekly series is densified per (stream, symbol, user) from the user's first observed week to the last complete week in the write window, so `ROWS BETWEEN 51 PRECEDING AND CURRENT ROW` represents a true 52-calendar-week trailing sum. Week key convention: `toStartOfWeek(date, 1)` (Monday-start), matching the weekly-mart convention used throughout the repo. Memory strategy: the query is split per (stream_type, symbol) and UNION ALL-ed so each aggregate / sort / window-function state is bounded by a single stream-symbol's user population instead of the cross-product of every symbol × every user at once. An optional `slice` var (comma-separated `stream:SYMBOL` pairs, or a bare `SYMBOL`) selects which slices to process per run; staged batches in schema.yml use it. Incremental strategy: - Plain incremental: recompute the trailing `lookback_weeks` weeks; the upstream daily filter pulls `window_history_weeks` of prior source data so the rolling window has its full 52-week history for every rewritten week. - Batch refresh (`start_month` / `end_month`, driven by scripts/full_refresh/refresh.py): write only weeks whose `toStartOfMonth(week)` falls in [start_month, end_month]. Daily feed reaches `window_history_weeks` before `start_month` and extends 6 days past the last day of `end_month` so any ISO week that starts in `end_month` but finishes in the next calendar month is summed from all 7 of its daily rows; the next batch's start_month filter rejects that week's `toStartOfMonth`, preventing double-write.
