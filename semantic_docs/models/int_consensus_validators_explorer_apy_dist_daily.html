int_consensus_validators_explorer_apy_dist_daily

Per-(date, withdrawal_credentials) cross-sectional APY distribution plus rolling 7d/30d medians of the credential's balance-weighted daily APY. Built on int_consensus_validators_income_daily INNER JOIN fct_consensus_validators_status_latest with the exit-date guard from fct_consensus_validators_explorer_daily. Two smoothings coexist: * Cross-sectional quantiles (q05_apy..q95_apy) across the validators in the credential's set — meaningful for multi-validator credentials, collapsed for solo credentials. * Time-series rolling median (apy_rolling_7d_median, apy_rolling_30d_median) on the credential's own daily weighted-mean APY — the primary line the dashboard plots for solo credentials where the raw daily APY is jittery. Incremental monthly with a 30-day lookback so the rolling median at partition boundaries is seeded correctly. Order (withdrawal_credentials, date) so the API filter prunes via the primary index.
