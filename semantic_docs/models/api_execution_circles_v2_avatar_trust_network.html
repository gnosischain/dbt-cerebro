api_execution_circles_v2_avatar_trust_network

Trust-network edge list for the Circles v2 Avatar Trust Network panel. One row per directed trust edge from the perspective of the focal avatar. For mutual relationships TWO rows are emitted (avatar → counterparty AND counterparty → avatar) so the chart renders both arrows. The dashboard's avatar global filter scopes rows to one focal avatar — ClickHouse pushes that filter down into each UNION branch, so per-request cost stays small despite the apparent size of the query. Materialising this globally would produce ~millions of rows × long IPFS strings and blow the memory budget, so this stays a view on purpose. Each row carries display name and IPFS preview image for both endpoints (joined from api_execution_circles_v2_avatar_metadata). The dashboard graph chart consumes these via sourceImageField / targetImageField to render avatar thumbnails on the nodes. The target_layer / source_layer columns drive the chart's concentric layout: "Focal avatar" lives at the centre, "Mutual" forms the inner ring, "Trust given" the middle ring, "Trust received" the outer ring.

Columns

NameTypeDescription
avatarStringThe focal avatar this row belongs to (used by the dashboard's avatar global filter).
directionStringOne of "Mutual" / "Trust given" / "Trust received" — drives the edgeStyleField on the dashboard graph panel so each direction renders in its own colour. Mutual relationships emit two rows (one per direction) so both arrows render.
source_idStringAddress of the truster on this edge.
source_imageStringIPFS preview image URL of the source endpoint, or empty when no profile image is published.
source_layerStringConcentric ring assignment for the source endpoint. "Focal avatar" when the source IS the focal avatar (centre), otherwise the direction label (Mutual / Trust given / Trust received) so the counterparty lands on the correct ring.
source_nameStringDisplay name of the source endpoint (IPFS profile name → on-chain name → truncated address).
target_idStringAddress of the trustee on this edge.
target_imageStringIPFS preview image URL of the target endpoint, or empty when no profile image is published.
target_layerStringConcentric ring assignment for the target endpoint. "Focal avatar" when the target IS the focal avatar (centre), otherwise the direction label.
target_nameStringDisplay name of the target endpoint (IPFS profile name → on-chain name → truncated address).
valueUInt8Constant 1 (trust edges are unweighted; the column exists so the chart engine can size nodes by degree).
