api_execution_live_trades

Dashboard-facing feed of recent DEX trades on Gnosis Chain. Reads from the cached `int_live__dex_trades_raw` table, so user queries never hit source logs or trigger decode-log scans. Last 30 minutes of CACHED data (anchored on `max(block_timestamp)` in that table, which trails the live source by the incremental refresh cadence) with a 60-second reorg buffer. Multi-hop router swaps are collapsed to one row per transaction. Dust is filtered via `live_trades_min_usd` var (default 1 USD); rows with unknown USD are kept. No LIMIT is applied — dashboards should add their own. For ingestion-level staleness, query `api_execution_live_trades_freshness` separately.
