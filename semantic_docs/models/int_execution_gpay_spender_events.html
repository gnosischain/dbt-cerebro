int_execution_gpay_spender_events

Decoded events from the Gnosis Pay Spender module proxies attached to GP Safes. The Spender is GP's custom final gatekeeper for card-spend authorization (deployed and controlled by GP's settlement infra). Events of interest: Spend(asset, account, receiver, amount) — THE event. Fired on every successful card transaction. The headline analytics signal for GP card activity. asset — token being spent (ERC-20 contract address) account — the GP Safe being debited receiver — settlement / merchant-acquirer destination amount — token base units (string; could be large) AvatarSet(previousAvatar, newAvatar) — Spender bound to a Safe (or rebound). The newAvatar is the GP Safe this Spender module belongs to. Setup signal. TargetSet(previousTarget, newTarget) — The Spender's target contract changed. EnabledModule / DisabledModule — Sub-modules of the Spender enabled or disabled (the Spender itself acts as a module manager for its own sub-stack). ExecutionFromModuleSuccess / ExecutionFromModuleFailure — A sub-module forwarded a call through the Spender. OwnershipTransferred / Initialized — Standard ownable / proxy lifecycle. HashExecuted / HashInvalidated — Signed-message tracking. The parameter is an unnamed bytes32 so we cannot pluck it by key — we just record the event fired and leave the bytes32 in the raw decoded_params for downstream consumers that need it.
