int_execution_gnosis_app_swaps

Gnosis App swaps — one row per CoW order pre-signed by a GA user through the Cometh ERC-4337 bundler. Matches the Dune semantic: the anchor is the PreSignature event, NOT the Trade fill. Orders that are pre-signed but never filled (cancelled, expired, solver mismatch) are still counted as app activity. Strict heuristic (all must hold for a row to appear): 1. GPv2Settlement.PreSignature event with signed=true 2. PreSignature tx.to = ERC-4337 EntryPoint v0.7 (0x…7727de22…) 3. PreSignature tx.from ∈ Cometh Relayers V4 (seed gnosis_app_relayers) 4. PreSignature owner ∈ GA users (int_execution_gnosis_app_users_current) Trade enrichment (tokens, amounts, USD, solver) is a LEFT JOIN — NULL for unfilled orders. `was_filled` surfaces this as a boolean. Partial fills produce multiple Trade rows for one PreSignature; for the LEFT JOIN we pick the first fill (argMin on block_timestamp) and aggregate amounts via sum so one PreSignature = one output row. Grain: (transaction_hash, log_index) of the PreSignature event. Row count on this model = count of GA-app signed orders.
