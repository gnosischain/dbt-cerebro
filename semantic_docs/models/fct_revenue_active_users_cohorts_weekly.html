fct_revenue_active_users_cohorts_weekly

HEADLINE — cross-stream unique-active-user cohort (rolling 52-week). For each user, fees across ALL streams (holdings + sDAI + GPay) and ALL symbols are SUMMED FIRST, then the total is bucketed into a $-cohort. Because aggregation happens before bucketing, a user cannot appear in multiple cohorts even if they accrue fees in several sectors. This is the canonical cross-sector "active user" breakdown by economic size; the `$1+`/`$6+`/... buckets are annualised. See file header for the active-user definition.
