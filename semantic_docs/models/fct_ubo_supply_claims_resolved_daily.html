fct_ubo_supply_claims_resolved_daily

One-pass second-level-container resolution over fct_ubo_supply_claims_daily. Identical schema and unique key. For any row where ubo_address is itself a known container for the bridge token (bridge token = container_address, which equals the share-token address for ERC4626 vaults), that row is replaced by the container's own sub-holders redistributed proportionally. All other rows pass through unchanged. If the same ubo appears via both a direct path and a redistributed path, balances are summed. Downstream consumers (fct_execution_tokens_top_holders_ranked, _latest, ubo_coverage_latest) use this model instead of the raw fct.
