Columns

NameTypeDescription
active_vs_heuristic_ratioRatio of the canonical GT active-user count to the heuristic user count: round(gt_registered_active / heuristic_users, 4). >1 means the GT active set is larger than what the heuristic captures (sizing the heuristic's under-count vs ground truth); can exceed 1. Gotcha: unguarded division -> NaN/Inf if heuristic_users = 0. Unit: ratio (dimensionless).
avatar_containmentheuristic_in_avatar / heuristic_users (expected ~1.0).
gt_active_in_heuristicNumber of canonical GT active users that the legacy address heuristic ALSO sees: countIf(is_registered_active AND is_heuristic_active). is_heuristic_active (upstream) = identity present in int_execution_gnosis_app_users_current. Measures the overlap/recall of the heuristic against the GT active set. Unit: user count.
gt_active_legacy_onlyNumber of canonical GT active users whose activity is purely LEGACY (Metri) with no current-app footprint: countIf(is_registered_active AND legacy_app_signal AND NOT current_app_signal). Upstream legacy_app_signal = has_swap_metri OR has_metri_fee OR has_metri_transfer OR has_investment OR has_auto_topup; current_app_signal = has_swap_gnosis_io OR is_heuristic_active OR has_cashback OR has_pay_topup. Isolates users the current-app heuristic structurally misses because they only ever used the legacy app. Unit: user count.
gt_active_missed_by_heuristicNumber of canonical GT active users the current-app heuristic CANNOT see (blind spot, ~17.5k): countIf(is_registered_active AND NOT is_heuristic_active). These are largely legacy/non-current-app users invisible to the current-app-only heuristic. Complement of gt_active_in_heuristic within gt_registered_active. Unit: user count.
gt_avatar_usersTotal distinct Circles avatar identities in the envio_ga avatar table (stg_envio_ga__avatars = deduped envio_ga.avatar via envio_latest with composite version). This is the active-user CONTAINMENT SPINE for the heuristic (broader membership than the registry). Unit: identity count. Derived as (SELECT count() FROM stg_envio_ga__avatars).
gt_registered_activeCANONICAL GT active-user count (~26.6k): number of rows in int_execution_gnosis_app_gt_user_activity where is_registered_active is TRUE = a registered identity that performed at least one APP-TAGGED action (gnosis.io or Metri swap, MetriFee, MetriTransfer, pay-topup, cashback, investment, auto-topup config). This is the headline Gnosis App active-user metric. Unit: user count. Derived as countIf(is_registered_active).
gt_registered_active_incl_circlesBROADER (sizing-only, NOT headline) active-user count (~40k): rows where is_registered_active_incl_circles is TRUE = registered identity with any app-tagged action OR any GENERIC untagged Circles/social action (PersonalMint, HubTransfer, PrimaryGroupFee, Erc20WrapperTransfer/wrap, InvitationFee, referral earned/sent). Because generic Circles actions can originate from any Circles client (no app tag), this OVER-reaches 'used the Gnosis App'. Unit: user count. Derived as countIf(is_registered_active_incl_circles).
gt_registry_usersTotal distinct Circles identities in the envio_ga registry (stg_envio_ga__users = deduped envio_ga.gnosis_app_user via envio_latest). ~301k. This is the ecosystem-wide REGISTERED identity universe (one row per address, tombstoned ids removed), NOT a users/active metric. Unit: identity count. Derived as (SELECT count() FROM stg_envio_ga__users).
heuristic_in_avatarNumber of heuristic addresses that are also present in the GT avatar spine: countIf(addr IN (SELECT avatar_address FROM stg_envio_ga__avatars)) over the heur set. Numerator of avatar_containment; expected near-total (~99.98%) because the avatar table is the active-user containment spine. Unit: user count.
heuristic_in_registryNumber of heuristic addresses that are also present in the GT registry: countIf(addr IN (SELECT address FROM stg_envio_ga__users)) over the heur set. Numerator of registry_containment. Registry addresses are already lowercased upstream so the lower(address) heur side matches. Unit: user count.
heuristic_only_vs_registryHeuristic addresses NOT found in the GT registry: heuristic_users - heuristic_in_registry. The residual/uncontained tail of the heuristic relative to the registry (heuristic hits with no registered Circles identity). Unit: user count. Non-negative by construction.
heuristic_users
registry_containmentheuristic_in_registry / heuristic_users. Gate >= 0.90 (see tests/gnosis_app_gt_containment_gate.sql).
