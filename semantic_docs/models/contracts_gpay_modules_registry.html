contracts_gpay_modules_registry

Cross-referenced registry of every Zodiac module proxy that is currently or was ever enabled on a Gnosis Pay Safe. Two evidence sources, joined on the proxy address: 1. int_execution_safes_module_events — the Safe enabled this address as a module (filtered to GP Safes via int_execution_gpay_wallets). 2. int_execution_zodiac_module_proxies — the address was deployed by the canonical Zodiac ModuleProxyFactory and points at one of the three known GP mastercopies. The INNER JOIN means we drop any "enabled module" event that points to an address we don't recognise as a Zodiac proxy of a GP-relevant type. This is the high-quality registry that decode_logs consumes via contract_address_ref to decode per-module events downstream. Output schema matches the standard contracts_*_registry shape so decode_logs picks up abi_source_address automatically.
