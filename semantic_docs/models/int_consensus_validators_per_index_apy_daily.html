int_consensus_validators_per_index_apy_daily

Per-(date, validator_index) APY for active / pending-queued validators. Rescoped 2026-06: `apy` is now read straight from int_consensus_validators_income_daily (consensus base-reward cap + ledger-exact effective-credit math) instead of the old mod-32-GNO deposit-rounding trick, which broke on EIP-7251 (MaxEB) sub-32-GNO top-ups and produced per-validator APY up to ~1e47% and down to -100% on consolidation / top-up days. Carries only the per-validator status + balance its distribution consumers need (int_consensus_validators_dists_daily, fct_consensus_validators_dists_last_30_days); balance is in gwei.
