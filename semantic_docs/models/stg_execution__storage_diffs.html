stg_execution__storage_diffs

The stg_execution__storage_diffs model captures storage difference events from blockchain execution transactions, enabling analysis of state changes at the storage level during contract execution.

Module: execution

Status: docs_only

Owner: analytics_team

Path: models/execution/state/staging/stg_execution__storage_diffs.sql

Tags: production execution state size

Columns

NameTypeDescription
addressStringThe blockchain address involved in the storage difference, formatted with '0x' prefix.
block_numberUInt64The number identifying the block in which the transaction was included.
block_timestampDateTime64(0, 'UTC')UTC timestamp of the block when the storage change occurred.
chain_idUInt64EVM chain identifier (e.g., 100 for Gnosis).
from_valueStringHex-encoded value before the change.
insert_versionUInt64Monotonically increasing version number indicating ingestion order.
slotUInt64The specific storage slot affected by the transaction, indicating the location of the storage change.
to_valueStringHex-encoded value after the change.
transaction_hashStringThe unique hash identifier for the transaction, prefixed with '0x'.
transaction_indexUInt64The position of the transaction within the block, indicating its execution order.
