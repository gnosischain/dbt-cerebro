fct_execution_gpay_journeys_7d

Pre-joined GP-side journey spine with a 7-day lookback. One row per (conversion, prior-7d touchpoint) pair. Built via `build_journey_lookback(7, 'gpay')` — the same macro that powers the GA spine, with `identity_role` matched on both sides of the conversion ↔ events join so role-grain semantics are preserved (treasury-grain conversions match treasury-grain touchpoints, etc.). Leakage guards enforced at materialisation: * `e.event_ts < c.conversion_ts` * `e.identity_role = c.identity_role` * `e.event_kind != conversion_kind_to_event_kind(c.conversion_kind)` Microbatched in 1-month chunks via `scripts/full_refresh/refresh.py` (`meta.full_refresh.batch_months: 1`). A single 2-year full-refresh OOM'd at the 10 GiB per-query cap because of the 3-role fan-out × all GP activity × 7-day lookback. Per-month batches fit comfortably; the refresh script auto-retries the rare batch that pushes the limit.
