int_celo_gpay_safe_current_owners

Current owner set per Safe, folded directly from int_celo_gpay_wallet_events. Mirrors Gnosis Chain's int_execution_safes_current_owners fold pattern exactly. No synthetic seed row needed (unlike an earlier version of this model) — issued_at is a real ingested event here, not derived separately, so a Safe with no add/remove events still naturally has its initial owner represented. This is a SEPARATE, additive signal from int_celo_gpay_wallets — that model's owner_address intentionally stays the initial SafeSetup owner forever, for identity purposes, unaffected by anything tracked here. This model answers a different question ("who can currently sign for this Safe") and can legitimately show the dead sentinel address 0x0000000000000000000000000000000000000002 as the current owner for Safes where GP's provisioning flow disabled owner-based signing — that is not a bug, it correctly reflects the on-chain state.
