api_consensus_validators_search

Dropdown source for the Validator Explorer tab. Grain is one row per WITHDRAWAL_CREDENTIALS (not per validator) — on Gnosis today this is ~3,400 rows vs 558k at validator-grain, so the dashboard can load the full list in a single request (a few hundred KB on the wire) without 30s timeouts. Collapsing to credential-grain matches the Explorer tab's semantics: every chart on that tab aggregates across all validators sharing the selected withdrawal_credentials. A solo validator is just a "credential with validator_count = 1"; an operator is "credential with validator_count = N". display_name is shaped as '<prefix-4>…<address-tail-10> · N validator(s) (v#<first-index>)' using the tail of the credential (the actual execution address) so operators with the same 0x01 type-byte prefix remain visually distinguishable. LabelSelector filters client-side on substring match against every column the API returns, so pasting a withdrawal_credentials string from a block explorer or wallet finds the exact row. Pasting a validator_index or pubkey not in this column won't match here — those lookups need /api/validators_performance_latest, which stays at per-validator grain.
