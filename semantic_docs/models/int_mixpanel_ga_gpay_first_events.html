int_mixpanel_ga_gpay_first_events

INTERNAL ONLY. One row per Gnosis Pay account (identity_role=initial_owner) per first-event type. event_type is 'funded' (first gpay_funded inflow) or 'first_payment' (first gpay_payment). UTM is resolved via two paths, GA-controller wins: for app-onboarded Safes the initial_owner that signed Safe setup is the Cometh relayer (never matches Mixpanel), so we prefer the campaign of the Gnosis App user enabled on the Safe's Delay module (int_execution_gnosis_app_gpay_wallets.first_ga_owner_address, pseudonymized), falling back to the direct initial_owner pseudonym match, then 'unknown'. Only the campaign assignment changes; the account count per event_type is unchanged. Accounts with no signal get 'unknown' so totals reconcile to the full GP funnel. Carries user_pseudonym — never exposed to cerebro-api or MCP.
