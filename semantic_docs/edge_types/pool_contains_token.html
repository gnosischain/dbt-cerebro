descriptionIndividual LP add/remove/collect events across all protocols on Gnosis Chain. One row per (event, token) — not pivoted to wide format because Balancer pools can hold 2–8 tokens, making a fixed-width schema incorrect. Each protocol's event decoding lives in its own staging model (stg_pools__dex_liquidity_*). This model unions them, enriches with token metadata, adds USD prices (ASOF join), and joins execution.transactions for tx_from/tx_to. The tx join uses an IN subquery so ClickHouse builds a small hash set of event tx hashes and probes the large transactions table, rather than loading the full right side into memory. Protocols: Uniswap V3, Swapr V3, Balancer V2, Balancer V3. event_type values: 'mint' (add), 'burn' (remove), or 'collect' (claim fees; V3 only). amount_raw is always positive; event_type conveys direction. Includes V3 tick range (tick_lower/tick_upper) and signed liquidity delta for position-level tracking and in-range analysis. When called with start_month/end_month vars each protocol model filters to that month window; otherwise falls back to the incremental filter.
