validator_operators_distinct

Distinct count of EVM withdrawal addresses controlling 0x01-credential validators. The `user_pseudonym` join key is in the same hash space as revenue / gpay / gnosis_app / Circles — use cross-sector joins on `user_pseudonym` to compute overlaps (e.g. "validator operators who are also gpay users", "revenue-active users who run validators"). Excludes 0x00-credential validators (BLS-only, no EVM withdrawal address to hash).
